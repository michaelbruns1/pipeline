Oxygen Webhelp

Information Architects need a way to publish their content. Oxygen utilizes the DITA-OT plugin for just that need!

One of the publishing options provided within Oxygen is a Webhelp HTML5 output specifically designed for easy and professional looking publishing.

Oxygen has a set of publishing templates built for transformation scenerios. Some of the included senerios available are included as a collection of customizable Webhelp pages.

If prefered, there are a lot of customization that one can access through the parameters and configuration options available. These are base templates that can be used to help you along your way to creating a final output.

Descriptor File

In order to customize your Webhelp output, you need to make use of a descriptor configuration file. It has the .opt file extension. This allows you to do everything from controlling the transformation process and controlling the output.
  • Add custom logo and favicon
  • Show / hide components
  • Configure components
  • Configure various aspects of the topic content
  • Configure built-in search engine
  • Replace built-in search engine with a custom one
  • Configure generated site-map
The format of the descriptor file can be seen here:
Template Descriptor File examplehttps://www.oxygenxml.com/doc/versions/25.1/ug-editor/topics/whr_publishing_template_contents.html#ariaid-title2
<publishing-template>
    <name>Flowers</name>

    <webhelp>
        <tags>
            <tag>tree</tag>
            <tag>light</tag>
        </tags>
        <preview-image file="flowers-tree.png"/>
        
        <!-- Resources (CSS, favicon, logo and others) -->
        <resources>
            <!-- Main CSS file -->
            <css file="flowers.css"/>
            
             <!-- Resources to copy to the output folder -->
            <fileset>
                <include name="resources/**/*"/>
                <exclude name="resources/**/.svn"/>
                <exclude name="resources/**/.git"/>
            </fileset>
        </resources>
        
        <parameters>
            <parameter name="webhelp.show.main.page.tiles" value="no"/>
            <parameter name="webhelp.show.main.page.toc" value="yes"/>
            <parameter name="webhelp.top.menu.depth" value="3"/>
        </parameters>
    </webhelp>
</publishing-template>
Let's say you wanted to include tiles in your output, you did not want a table of contents, you wanted the top menu items to be activated when clicked, and you wanted the full table of contents links listed on the left hand side, you would simply update the parameters to reflect that need:
<parameter name="webhelp.show.main.page.tiles" value="yes" />
<parameter name="webhelp.show.main.page.toc" value="no" />
<parameter name="webhelp.top.menu.actived.on.click" value="yes" />
<parameter name="webhelp.default.collection.type.sequence" value="yes" />
<parameter name="webhelp.publication.toc.links" value="all" />

WebHelp Responsive Macros

You can use the whc:macro layout component to specify a macro value (a variable that will be expanded when the output files are generated).

A macro has the following syntax:
${macro-name}
or
${macro-name(macro-parameter)}
